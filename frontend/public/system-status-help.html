System Status Help Guide

 

This guide explains each System Status indicator and provides clear steps for resolving issues.

 

✅ Ollama Not Installed

Meaning: ARC‑NEXUS cannot detect the Ollama application on your system.

Fix: Install or Verify Ollama

Step 1 (Required): Download and install Ollama from: https://ollama.com/download

After installation, restart ARC‑NEXUS. No additional setup is required.

 

Step 2 (Optional): If Ollama was installed in a non‑standard location, reinstall it using the default installer location. This ensures your system recognizes it automatically.

Avoid installing Ollama inside your ARC‑NEXUS project folder.

 

✅ Ollama Not Running

Meaning: Ollama is installed, but the background service is not currently active.

Option 1: Start Ollama Manually (Recommended)

Steps:

  1. Open your console (Command Prompt, PowerShell, or Terminal)
  2. Run: ollama serve
  3. If the console shows 'listening' or stays open without errors or negative replies, Ollama is running correctly..

 

Option 2: Start the Ollama Service

Windows Steps:

  1. Open Services
  2. Locate Ollama
  3. Click Start

macOS Steps:

  1. Open Terminal
  2. Run: launchctl start io.ollama

 

✅ Config Not Ready

Meaning: ARC‑NEXUS cannot find or read its configuration file.

Option 1: Auto‑Generate a New Config (Recommended)

Steps:

  1. Click Fix Config in the System Status panel
  2. Wait for confirmation
  3. ARC‑NEXUS will create a clean, valid config file automatically

 

Option 2: Manual Reset

Steps:

  1. Open your user directory
  2. Delete the file: ~/.arc-nexus/config.json
  3. Restart ARC‑NEXUS
  4. A fresh config file will be created automatically

 

✅ Models Not Ready

Meaning: Required AI models are not downloaded or not available to Ollama.

 

Option 1: Auto‑Download Models (Recommended)

Steps:

  1. Click Fix Models in the System Status panel
  2. Wait for the download to complete
  3. ARC‑NEXUS will verify the models automatically

 

Option 2: Manual Model Pull

Steps:

  1. Open your console
  2. Run: ollama pull llama3.2 (or whichever model your ARC‑NEXUS configuration specifies)
  3. Wait for the download to finish
  4. Restart ARC‑NEXUS if needed
