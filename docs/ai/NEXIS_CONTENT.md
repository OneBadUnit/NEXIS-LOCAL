\# NEXIS PROJECT CONTEXT



\## Project Identity



NEXIS is a local-first AI-powered information workflow platform built under ARC NEXUS.



Core workflow:

1\. Collect

2\. Select

3\. Create

4\. Refine

5\. Reuse



NEXIS is designed to help users transform raw information into structured, reusable output.



The platform prioritizes:

\- practical workflows

\- maintainability

\- clarity

\- controllable AI integration

\- local-first flexibility



\---

\- MVP stabilization

\- shipping a stable usable product

\---



\# Development Philosophy



NEXIS intentionally favors:

\- practical engineering

\- iterative refinement

\- human-reviewed AI assistance

\- explicit logic

\- readable structure

\- maintainable code

\- controlled complexity



The project is NOT intended to become:

\- an enterprise framework

\- a microservice platform

\- a highly abstract architecture experiment



\---



\# AI-Assisted Development Workflow



Current workflow:

1\. AI reviews code

2\. Human validates recommendations

3\. Changes are manually applied

4\. Behavior is tested

5\. Changes are committed after verification



AI suggestions are advisory only.



The system should:

\- prefer evidence-based recommendations

\- avoid speculative rewrites

\- preserve working behavior

\- prioritize runtime correctness
